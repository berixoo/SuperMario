# 超级马里奥风格小游戏设计说明

## 1. 项目主题

本课程作业计划制作一款“超级马里奥风格”的横版平台跳跃小游戏。玩家控制一个原创红帽小英雄，在横向卷轴关卡中移动、跳跃、收集金币、躲避或踩踏敌人，并最终到达终点旗。

本项目只参考平台跳跃游戏的通用玩法，不使用官方《超级马里奥》的角色形象、地图、音乐或商标素材。项目内素材均为原创像素风资源，适合课程展示和学习使用。

## 2. 设计目标

1. 做出一个可以运行、可以通关的 Python 小游戏。
2. 体现 Python 基础知识：变量、函数、类、列表、集合、循环、条件判断、文件组织。
3. 体现游戏开发基础：游戏主循环、键盘输入、碰撞检测、角色状态、分数系统、音效播放。
4. 注释详细，便于老师检查和同学答辩。
5. 模块划分清晰，避免所有代码都写在一个文件里。

## 3. 游戏玩法

### 3.1 玩家操作

- `A` / `←`：向左移动
- `D` / `→`：向右移动
- `Space` / `↑`：跳跃
- `R`：重新开始

### 3.2 核心规则

- 玩家受重力影响，会自然下落。
- 玩家踩到地面或平台后可以再次跳跃。
- 玩家碰到金币后获得分数，金币从地图中消失。
- 玩家从上方落下并碰到敌人时，敌人被击败，玩家反弹。
- 玩家从侧面碰到敌人时损失生命。
- 玩家生命归零后游戏结束。
- 玩家到达终点旗后通关。

### 3.3 分数建议

- 收集 1 枚金币：+100 分
- 踩败 1 个敌人：+300 分
- 通关奖励：可按剩余生命或剩余时间额外加分

## 4. 场景与界面

### 4.1 开始界面

显示游戏名称、操作说明和开始提示。课程作业中可以简化为“按任意键开始”。

### 4.2 游戏界面

界面包含：

- 背景图
- 地面和平台砖块
- 玩家角色
- 金币
- 敌人
- 终点旗
- 顶部 HUD：分数、生命、金币数量

### 4.3 结束界面

游戏结束或通关时显示结果，并提示按 `R` 重新开始。

## 5. 地图设计

建议使用文本地图描述关卡，每个字符表示一个格子：

```text
# = 地面或砖块
P = 玩家出生点
C = 金币
E = 敌人
F = 终点旗
. = 空白
```

示例：

```text
....................F
.......C..........###
.....#####...........
P.........E......C...
#####################
```

这种方式的优点是简单直观，不需要额外地图编辑器，老师也能直接看懂关卡结构。

## 6. 技术方案

### 6.1 游戏主循环

Pygame 游戏一般采用固定流程：

1. 读取用户输入
2. 更新玩家、敌人、金币等游戏状态
3. 判断碰撞和胜负
4. 绘制画面
5. 控制帧率

### 6.2 碰撞检测

使用矩形碰撞检测。每个角色、敌人、金币、砖块都有一个矩形区域。只要两个矩形重叠，就说明发生碰撞。

玩家和砖块的碰撞建议分成横向和纵向处理：

- 横向移动后检查是否撞墙
- 纵向移动后检查是否落地或顶到砖块

这样可以减少穿墙、卡墙和地面抖动问题。

### 6.3 摄像机滚动

地图宽度大于窗口宽度时，可以让摄像机跟随玩家移动。绘制时不改变真实坐标，只把世界坐标减去摄像机偏移量，得到屏幕坐标。
