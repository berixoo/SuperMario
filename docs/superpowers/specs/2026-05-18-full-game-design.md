# 超级马里奥风格小游戏 — 完整版设计文档

Date: 2026-05-18

## 1. 概述

基于 Pygame 2.x 的横版平台跳跃游戏，目标是完成一个可提交、可演示、可由小组协作开发的完整版课程项目。游戏包含 3 个关卡、完整菜单系统、问号砖块、倒计时、最高分存档和原创像素风素材。

本项目参考经典平台跳跃玩法，但不直接使用官方《超级马里奥》的角色、地图、音乐或商标素材。所有图片和音频素材放在 `assets/` 目录，作为课程学习和展示使用。

## 2. 技术栈

- Python 3.10+
- Pygame 2.x
- 文本关卡地图
- JSON 最高分存档
- GitHub 仓库用于小组协作和队友下载代码；游戏运行本身不依赖 GitHub 或 gh CLI。

## 3. 模块架构

```text
main.py
  ↓
game.py          ← 主循环、输入、绘制、音效、状态机
  ↓        ↘
core.py    sprites.py  ← 纯规则数据 / Pygame 显示封装
  ↓
level_data.py  ← 3 个关卡文本地图
  ↓
config.py      ← 全局常量、路径、颜色
assets/        ← 图片 + 音频
save/          ← 运行时生成的最高分存档
```

- `main.py`: 入口，调用 `game.main()`。
- `config.py`: 窗口、物理参数、素材路径、颜色、帧率。
- `level_data.py`: 3 个关卡的文本地图字符串列表。
- `core.py`: 纯规则数据和逻辑，包括碰撞检测、重力、分数、生命、关卡进度；不依赖 Pygame，不负责加载图片。
- `sprites.py`: Pygame 显示封装，把 `core.py` 中的规则对象绘制成图片，并处理动画帧；不重复保存分数、生命等核心状态。
- `game.py`: 游戏主循环、状态机、输入、绘制、音效、存档读写调度。

## 4. 状态机

```text
TITLE → LEVEL_SELECT → PLAYING ↔ PAUSED
  ↑          ↑             ↓
  └──── GAME_OVER      WIN ─┴→ LEVEL_SELECT / NEXT_LEVEL / TITLE
```

| 状态 | 说明 |
|------|------|
| TITLE | 标题画面，游戏名 + 操作说明 + “按任意键开始” |
| LEVEL_SELECT | 3 个关卡卡片，已解锁可选，显示最高分 |
| PLAYING | 主游戏循环 |
| PAUSED | ESC 暂停，继续/重新开始/返回菜单 |
| WIN | 通关画面，分数 + 通关时间，可进入下一关 |
| GAME_OVER | 失败画面，最终分数，按 R 重来或回菜单 |

状态转移规则：

- `TITLE` 按任意键进入 `LEVEL_SELECT`。
- `LEVEL_SELECT` 只能选择已解锁关卡，默认第 1 关解锁。
- `PLAYING` 按 `ESC` 进入 `PAUSED`，暂停菜单可继续、重开本关或返回选关。
- `WIN` 如果不是第 3 关，可选择进入下一关；如果是第 3 关，显示全通关提示并返回选关或标题。
- `GAME_OVER` 可按 `R` 重开当前关，也可返回标题或选关。
- 通关某一关后，下一关解锁，并写入存档。

## 6. 游戏机制

### 6.1 玩家物理

- 移动速度: 300 px/s
- 跳跃初速度: -720 px/s
- 重力: 1800 px/s²
- 最大下落速度: 960 px/s
- 所有运动使用 `dt`（秒）计算，避免帧率变化导致游戏速度变化。
- 横向/纵向碰撞分开处理，先横后纵。
- 不支持二段跳，仅落地可跳。

### 6.2 碰撞规则

| 碰撞 | 效果 |
|------|------|
| 金币 | 金币消失，+100 分，播放 `coin.wav` |
| 敌人（侧碰） | 玩家受伤，-1 命，1.5s 无敌闪烁，播放 `hurt.wav` |
| 敌人（踩踏） | 敌人死亡，+300 分，玩家反弹，播放 `jump.wav` |
| 问号砖块（顶击） | 随机出金币（+100 分）或加命（+1），砖块变为 `U` 已触发状态 |
| 终点旗 | 通关，显示通关结算 |

### 6.3 敌人 AI

- 蘑菇敌人：左右巡逻，遇墙或平台边缘转向，速度 90 px/s。
- 飞行敌人：水平巡逻并按 sin 波形上下浮动，速度 120 px/s，浮动幅度 24 px。

### 6.4 生命与倒计时

- 初始 3 条命。
- 生命归零 → `GAME_OVER`。
- 时间归零 → 扣 1 命，当前关卡重置。
- 玩家掉出地图底部 → 扣 1 命，并从当前关卡出生点重生。
- 倒计时：第 1 关 120s，第 2 关 100s，第 3 关 90s。

### 6.5 HUD

- 左上：分数。
- 右上：生命数量。
- 中上：倒计时。
- 分数旁显示金币计数。

## 7. 问号砖块

- 玩家从下方顶击触发。
- 随机：50% 金币（+100 分），50% 加命（+1）。
- 触发后砖块变为已使用灰色砖块，不可再次触发。
- 像素动画：砖块被顶起轻微弹跳。
- 顶击奖励如果生成实体道具，优先生成在砖块上方；如果空间不足，则直接结算奖励。

## 8. 最高分存档

- 存储位置：`save/highscore.json`。
- 首次运行如果文件不存在，则自动创建默认存档。
- 格式：

```json
{
  "unlocked_level": 1,
  "best_scores": {
    "level_1": 0,
    "level_2": 0,
    "level_3": 0
  }
}
```

- 每关通关时与最高分比较，更新并写入文件。
- 通关第 1 关解锁第 2 关，通关第 2 关解锁第 3 关。
- 关卡选择界面显示各关最高分和锁定状态。
- 存档写入失败时，游戏继续运行，但在界面上提示“最高分保存失败”。

## 9. 音频

- 背景音乐：`theme_loop.wav` 循环播放。
- 跳跃：`jump.wav`，起跳时播放。
- 金币：`coin.wav`，收集金币时播放。
- 受伤：`hurt.wav`，碰敌人受伤时播放。
- 音频设备不可用时静默运行，不崩溃。
