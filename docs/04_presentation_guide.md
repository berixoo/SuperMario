## 1. 项目概述 (30 秒)

### 讲什么

"本项目实现了一个超级马里奥风格的横版平台跳跃游戏。
使用 Python 3.10 + Pygame 2.6 开发，包含 3 个关卡、6 种游戏状态、2 种敌人 AI、问号砖块随机奖励系统和最高分存档。

项目特点是模块化设计：将游戏规则与界面渲染分离，
核心逻辑模块 (core.py) 不依赖 Pygame，可以独立进行单元测试。"

### 示范操作

- 打开项目目录，展示文件结构
- 运行 `python main.py` 进入标题画面

### 关键数据

- 6 个 Python 文件，~1200 行代码
- 33 条单元测试，覆盖率核心逻辑
- 3 个关卡 (120 秒)，难度递进
- 原创像素风素材和 8-bit 音频


## 2. 模块架构 (1 分钟)

### 架构图 (讲解时可用 PPT 展示)

```
main.py          ← 入口，一行启动
  ↓
game.py          ← 主循环、状态机、输入、音效
  ↓        ↘
core.py    sprites.py  ← 规则/显示分离
  ↓
level_data.py    ← 文本地图
  ↓
config.py        ← 全局常量
assets/          ← 素材
```

### 讲什么

"项目按照单一职责原则分为 6 个模块：

- **main.py** 是入口，只负责创建 Game 并调用 run()
- **config.py** 集中管理所有可调参数：窗口大小、物理常量、颜色、路径。修改游戏速度只需改一个数字
- **level_data.py** 用文本字符串存储 3 个关卡地图，`#` 代表砖块、`P` 代表玩家、`E` 代表敌人。这种设计让关卡结构一目了然，不需要地图编辑器
- **core.py** 是纯 Python 规则模块，不导入 Pygame。定义了 Rect、Player、Enemy 等实体类，以及碰撞检测、物理更新、AI 行为等函数。可以直接用 pytest 测试，不需要启动游戏窗口
- **sprites.py** 负责所有 Pygame 绘制：加载图片、缩放、绘制角色和 HUD
- **game.py** 是总控制器，管理 6 种状态的流转和主循环"

### 讲解要点

"这种分离的好处是：想改游戏规则，只改 core.py；想改画面效果，只改 sprites.py；
两者互不影响。答辩时如果老师问'碰撞检测在哪'，直接打开 core.py 即可。"


## 3. 状态机演示 (1 分钟)

### 状态流转图

*操作: 按 1 开始游戏*

**(3) 暂停 → 继续** (15s)

"游戏中按 ESC 暂停，显示半透明遮罩和菜单。可以继续、重开或返回选关。"

## 4. 核心玩法演示 (1.5 分钟)

### 边玩边讲 (选择第 3 关可以展示更多机制)

**(1) 物理系统** (30s)

"玩家受重力影响自然下落。移动和跳跃都乘以 dt (帧间隔)，保证不同帧率下游戏速度一致。
dt 上限 0.05 秒，防止卡顿时一帧跳太远导致穿墙。

碰撞检测使用分离轴算法：先把水平移动和纵向移动分开处理，
先检查横向撞墙，再检查纵向落地或顶头。这样不会出现卡在墙角的问题。"

*操作: 左右移动、跳跃、落地*

**(2) 金币收集** (15s)

"碰到金币后金币消失，加 100 分。收集数量显示在 HUD 上。"

*操作: 走到金币附近收集*

**(3) 敌人 AI 和碰撞** (30s)

"蘑菇敌人会在地面和平台上左右巡逻，遇到墙或平台边缘自动转向。
它们受重力影响，会下落到平台上。

玩家从上方落到敌人头上 → 踩踏成功，敌人死亡，+300 分，玩家弹起。
从侧面碰到敌人 → 玩家受伤，-1 命，1.5 秒无敌闪烁保护。"

*操作: 踩一个敌人，再演示受伤*

**(4) 问号砖块** (15s)

"第 3 关特有。从下方顶击问号砖块，50% 概率出金币，50% 概率加 1 条命。
触发后砖块变灰色，不可再次触发。砖块有弹跳动画。"

*操作: 跳到问号砖块下方顶击*


## 5. 关卡设计 (1 分钟)

### 展示关卡文本

打开 `super_mario/level_data.py`，展示 LEVEL_1 的前几行。

### 讲什么

"三个关卡使用文本字符串定义。每个字符对应一个 32×32 的瓦片：

- `#` 是砖块，`P` 是玩家出生点，`C` 是金币
- `E` 是蘑菇敌人，`B` 是飞行敌人
- `Q` 是问号砖块，`F` 是终点旗，`.` 是空白

这种方式的优势：
1. 直观 — 打开文件就能看到关卡全貌
2. 易改 — 加一个 `#` 就是平台，加一个 `E` 就是敌人
3. 适合课程展示 — 老师能直接看懂

三关难度递进：
- 第 1 关「绿色草原」：120 秒，简单跳跃 + 6 个蘑菇敌人
- 第 2 关「地下洞穴」：120 秒，新增 10 个飞行敌人 (B)，17 个敌人
- 第 3 关「天空城堡」：120 秒，新增 10 个问号砖块 (Q)，38 个敌人，44 个金币

阶梯设计：每关右端有阶梯式平台，逐级跳跃可达终点旗。
最大间距 2 格 (64px)，远小于玩家最大跳跃高度 4.5 格 (144px)。"


## 6. 代码亮点讲解 (1.5 分钟)

### 亮点 1: 规则与显示分离 (40s)

"打开 `super_mario/core.py`。这个文件不导入 Pygame，
所有的 Rect、Player、Enemy、碰撞检测函数都是纯 Python 逻辑。

好处是可以直接用 pytest 测试：
打开 `tests/test_core.py`，这里 33 条测试覆盖了
矩形碰撞、关卡解析、物理更新、金币收集、踩踏 vs 受伤判定、
问号砖块奖励、存档读写——全部可以在命令行运行，不需要启动游戏窗口。"

*操作: 运行 `python -m pytest tests/test_core.py -v`，展示 33 passed*

### 亮点 2: dt 帧率无关运动 (25s)

"在 `update_player` 函数中，所有运动都乘以 dt：
- 水平移动: `player.x += player.vx * dt`
- 重力加速: `player.vy += GRAVITY * dt`
- 跳跃初速: `player.vy = PLAYER_JUMP_VEL` (一次性赋值)

速度单位统一为 px/s，配合 dt 使用。
这样无论电脑跑 30fps 还是 60fps，玩家移动速度看起来都一样。"

### 亮点 3: JSON 存档与容错 (25s)

"最高分和关卡解锁进度持久化到 `save/highscore.json`。
存档读取有三层容错：
1. 文件不存在 → 返回默认值
2. JSON 解析失败 → 返回默认值
3. 旧版本存档缺少新字段 → 用默认值填充

写入失败不会导致游戏崩溃，只会在通关画面上显示'最高分保存失败'的提示。"


## 7. 总结 (30 秒)

### 讲什么

"本项目完整实现了一个可运行、可通关的平台跳跃游戏。

技术收获：
- 掌握了 Pygame 游戏开发的基本流程 (主循环、事件处理、碰撞检测)
- 实践了面向对象的模块化设计 (数据类、规则分离、状态机)
- 理解了帧率无关的游戏物理 (dt-based motion)
- 学会了用文本地图定义关卡内容

项目代码已开源在 GitHub，配有详细中文注释和单元测试，
可直接用于课程展示和后续扩展。"

## 答辩要点速查

| 可能被问到的问题 | 回答要点 |
|-----------------|---------|
| 为什么不用 Unity/Godot? | 课程要求 Python，Pygame 轻量、学习成本低 |
| 碰撞检测怎么做的? | AABB 矩形碰撞 + 分离轴处理 (先横后纵) |
| 怎么防止穿墙? | dt 上限 0.05s + 分离横纵轴碰撞修正 |
| 敌人 AI 怎么实现? | 蘑菇: 巡逻 + 重力 + 边缘检测转向; 飞行: sin 浮动 + 撞墙转向 |
| 问号砖块怎么触发? | update_player 在纵向碰撞中检测 isinstance(solid, QuestionBlock) |
| 不同帧率怎么保证? | 所有运动乘 dt, dt = clock.tick(FPS) / 1000.0 |
| 存档怎么存的? | JSON 文件, 三层容错 (文件不存在/解析失败/字段缺失) |
| 代码量多少? | ~1200 行 Python + 33 条单元测试 |
