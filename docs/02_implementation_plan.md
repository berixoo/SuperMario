# 超级马里奥风格小游戏实施文档

## 1. 开发环境

建议环境：

- Python 3.10 或以上
- Pygame 2.x
- Windows / macOS / Linux 均可

建议安装命令：

```bash
pip install pygame
```

## 2. 推荐目录结构

后续实现代码时，建议使用下面的结构：

```text
SuperMario/
├─ main.py
├─ requirements.txt
├─ super_mario/
│  ├─ __init__.py
│  ├─ config.py
│  ├─ core.py
│  ├─ level_data.py
│  ├─ sprites.py
│  └─ game.py
├─ assets/
│  ├─ images/
│  └─ audio/
└─ docs/
```

## 3. 开发步骤

### 第一步：搭建窗口

目标：

- 初始化 Pygame
- 创建窗口
- 设置标题
- 创建游戏主循环
- 控制帧率

验收标准：

- 运行 `python main.py` 后出现窗口
- 点击关闭按钮能正常退出

### 第二步：绘制背景和地图

目标：

- 加载 `assets/images/background_day.png`
- 用文本地图生成砖块
- 绘制地面、平台和终点旗

验收标准：

- 窗口中能看到背景和砖块
- 地图内容与文本关卡一致

### 第三步：实现玩家移动

目标：

- 加载玩家图片
- 实现左右移动
- 实现跳跃和重力
- 限制玩家不能穿过地面

验收标准：

- 玩家能左右移动
- 玩家能跳跃后落回地面
- 玩家不能掉进地面或穿过砖块

### 第四步：实现金币系统

目标：

- 绘制金币
- 玩家碰到金币后金币消失
- 分数增加
- HUD 显示分数

验收标准：

- 每个金币只能收集一次
- 分数显示正确

### 第五步：实现敌人系统

目标：

- 绘制蘑菇敌人
- 敌人自动左右移动
- 敌人遇到墙或平台边缘转向
- 玩家从上方踩到敌人时击败敌人
- 玩家从侧面碰到敌人时扣生命

验收标准：

- 敌人不会静止不动
- 踩敌人和受伤能区分
- 生命归零后进入失败状态

### 第六步：实现通关和重开

目标：

- 玩家碰到终点旗后显示通关
- 失败或通关后按 `R` 重新开始

验收标准：

- 通关状态不会继续扣血
- 重开后分数、生命、金币恢复初始状态

### 第七步：接入音效

目标：

- 背景音乐循环播放
- 跳跃、金币、受伤播放对应音效

验收标准：

- 游戏无音频设备时不崩溃
- 音效触发时机正确

## 4. 分工建议

如果小组有 3 人，可以这样分工：

1. 同学 A：游戏主循环、窗口、素材加载、界面绘制
2. 同学 B：玩家移动、重力、碰撞、摄像机
3. 同学 C：金币、敌人、分数、生命、文档整理

如果小组有 4 人，第 4 人可以负责关卡设计、素材整理、答辩 PPT 和测试。

## 5. 测试计划

建议至少检查以下内容：

| 测试项 | 检查方式 | 预期结果 |
|---|---|---|
| 窗口启动 | 运行主程序 | 游戏窗口正常出现 |
| 左右移动 | 按 A/D | 玩家水平移动 |
| 跳跃 | 按 Space | 玩家上升后下落 |
| 地面碰撞 | 跳到平台上 | 玩家站在平台上 |
| 金币收集 | 碰到金币 | 金币消失，分数增加 |
| 敌人踩踏 | 从上方落到敌人头上 | 敌人消失，玩家反弹 |
| 侧碰敌人 | 从侧面碰敌人 | 生命减少 |
| 通关 | 碰到终点旗 | 显示通关提示 |
| 重新开始 | 按 R | 游戏回到初始状态 |

## 6. 风险与解决方案

### 风险 1：玩家穿过地面

原因：一帧移动距离太大，直接越过砖块。

解决：把玩家移动拆成横向和纵向两步处理；必要时把大时间步拆成多个小时间步。

### 风险 2：代码写在一个文件里太乱

原因：课程小游戏容易从简单脚本慢慢变成大文件。

解决：从一开始就分模块：配置、核心规则、地图、精灵、主循环。

### 风险 3：官方素材版权问题

原因：直接下载官方马里奥图片和音乐可能不适合课程提交。

解决：使用本项目 `assets/` 中的原创像素风素材，并在答辩中说明素材来源。

### 风险 4：不同电脑声音播放失败

原因：机房电脑或远程环境可能没有音频设备。

解决：音频初始化失败时跳过音乐播放，但游戏主体继续运行。
